Github搭建Hexo博客教程

xiaoge Nov 06, 2016
  • 软件

前记

之前博客用的是wordpress程序,之后陆续见到了许多让我非常喜欢的个人博客.
发现它们都是基于Hexo搭建的,当然了,用的基本也是同一个主题,就动了迁移博客的念头.
刚好今天心情不咋好,书看不进去.就把这件事情做了.
官方教程地址()教程非常简洁.
确保拥有一个github账户,并新建一个repository.建议在电脑上安装github客户端
地址为desktop.github.com
在setting里设置一个username.github.io这个username一定要与github账户的用户名一致
(踩过这个坑)
点击绿色的clone下的小三角,点击clone,复制这个https地址,把它添加到deplyed下的url后面

1.准备工作

确保电脑可以科学上网.
安装git和node.js(就是一直不停地点下一步了.因为我之前某段时间曾经安装过git所以就不提了.(●˘◡˘●)
当时我看的教程是廖雪峰的Git教程,这一步必不可少,因为要用到跟)
查看git和node.js是否正确安装
cmd输入框分别输入git -v和node –v如果显示的是版本号,那么就证明安装正确了

2.操作说明

本地创建一个hexo文件夹(如果有同步盘的话建议装在同步盘里面,以防万一)
打开这个空白的文件夹,shift+鼠标左键,选择在此处打开命令窗口
输入$ npm install hexo-cli -g (g是全局的意思global)
随后在这个文件夹里就可以看到Hexo程序的基本结构
等文件夹

3.结构说明

source/_posts所有的博客文章都保存在下
themes 这里放的是博客的主题,从github下周主题后解压到这个文件夹下
(因为我的压缩软件的设置,在主题文件夹下又套了个同名文件夹才不无法显示,这是我踩过的坑)
_config.yml文件里保存的是配置文件
比如说博客语言,博客主题,
注意配置时,在每个:后面都要加一个空格再输入配置内容.

4.写博客

可以在markdown编辑器里直接编写文章,然后保存到source/_posts里,以.md结尾即可

5.发布

每次更新设置或者推送文章时,只要在命令行输入$ hexo g -d即可

6.更换主题

建议输入$ hexo clean 这个命令可以清除缓存和公共文件夹

7.从wordpress博客迁移过来

安装导入插件$ npm install wordpress –save
在wordpress后台,工具-导出
source为要导出的文件路径
导出后再$ hexo g -d一次,新建隐私窗口查看即可.

踩过的坑

因为某个原因系统重置了，hexo文件夹原来在/user/xxx/下的，怕文件丢失就放到了onedrive里自动
保存了。这是前提。
使用hexo时提示 hexo command not found。网上一番搜寻后找到的解决方法是：

卸载node.js。

重新安装node.js，并add到环境变量里。

再次初始化 hexo 就可以了。

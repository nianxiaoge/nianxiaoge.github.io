reset.css 重置浏览器的默认样式

xiaoge Dec 07, 2016
  • CSS
reset.css

IE,Firefox,Chrome,Safari.这些浏览器的默认HTML样式都不相同.例如p标签有上下边距,strong标签有字体
加粗样式,em标签有字体倾斜样式.不同浏览器的默认样式之间也会有差别,例如ul默认带有缩进的样式,在IE
下,它的缩进是通过margin实现的,而Firefox下,它的缩进是由padding实现的.解决方法就是重新设置浏览器的默认样式,也就是对于全局的样式定义.reset.css应运而生.

谷歌一番找到了两种主流方式,一种是reset.css一种是Normalize.css.

reset.css与Normalize.css的综述

reset 的目的是将所有的浏览器的自带样式重置掉,这样更易于保持各浏览器渲染的一致性.
normalize.css官网是这样描述normalize.css的:

Normalize.css is a small CSS file that provides better cross-browser consistency in the default
styling of HTML elements. It’s a modern, HTML5-ready, alternative to the traditional CSS reset.

翻译过来就是:Normalize.css 只是一个很小的CSS文件,但它在默认的HTML元素样式上提供了跨浏览器的
高度一致性.相比于传统的CSS reset,Normalize.css是一种现代的为HTML5准备的优质替代方案.
项目地址

创造这个项目的原因:

The aims of normalize.css are as follows:

Preserve useful browser defaults rather than erasing them.
Normalize styles for a wide range of HTML elements.
Correct bugs and common browser inconsistencies.
Improve usability with subtle improvements.
Explain the code using comments and detailed documentation.

译:
创造normalize.css有下面这几个目的:
保护有用的浏览器默认样式而不是完全去掉它们
一般化的样式:为大部分HTML元素提供
修复浏览器自身的bug并保证各浏览器的一致性
优化CSS可用性:用一些小技巧
解释代码:可用的注释和详细的文档

reset.css与Normalize.css的比较
normalize.css保留了部分可用的默认样式

reset.css为几乎所有元素统一了样式风格,但normalize保存了一些有用的默认样式,这就意味着你不用再为所有公共的排版元素重新设置样式.
当一个元素在不同的浏览器中有不同的默认值时,Normalize.css会力求让这些样式保持一致并尽可能与现代标准相符合.

normalize.css修复了一些bug

修复了常见的桌面端和移动端浏览器的bug.这往往超出了Reset所能做到的范畴.

normalize.css不会使你的调试工具混乱

使用Reset最让人困扰的地方莫过于在浏览器调试工具中大段大段的继承链.在Normalize.css中就不会有这样的问题.

normalize.css可以模块化

这个项目已经被拆分为多个相关却又独立的部分,这使得你能够很容易也很清楚地知道哪些元素被设置了特定的值.因此这能让你自己选择性地移除掉某些永远不会用到部分(比如表单的一般化).

normalize提供了详细的文档

Normalize.css的代码基于详细而全面的跨浏览器研究与测试.这个文件中拥有详细的代码说明并在Github Wiki中有进一步的说明.

如何使用 normalize.css

首先,npm安装npm install –save normalize.css
或从Github下载Normalize.css,有两种使用方法:

1:将normalize.css作为你自己项目的基本的CSS,自定义样式值以满足设计师的需求.
2:导入normalize.css,并适当修改.在必要的情况下用自己写的CSS覆盖默认值.

结束语

事实上,Normalize.css与Reset有很多不一样的地方,当你用在项目中时你就会发现.尝试使用之后选择适合你的偏好.
本项目在Github开源.任何人都可以提交问题报告或者补丁.每个人都可以看到这个项目的发展过程.
